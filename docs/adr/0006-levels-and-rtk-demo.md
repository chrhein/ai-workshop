# ADR 0006 — Behold nivå-oppsettet; rtk som demo; openclaw som plassholder

**Status:** Vedtatt · **Dato:** 2026-06-28

## Kontekst
Ny agenda er tema-basert (agent → skills → MCP → token → avansert), men forrige workshop hadde et nivå-oppsett som funket visuelt. 2 timer er stramt for 5 tema hands-on. rtk er fasilitatorens eget verktøy; openclaw-innhold er ikke klart.

## Beslutning
- **Behold nivå-strukturen** (4 nivå), men som **tema-progresjon**, ikke vanskelighetsgrad. Drop leverandør-bytteren ([ADR 0001](0001-claude-code-only.md)).
- **Token + avansert slås sammen til Nivå 4**, bevisst "mykt" så det kan kuttes ved tidsnød.
- **rtk presenteres som demo** ("X tokens vanlig vs. med rtk", `rtk gain"`), ikke obligatorisk installasjon for alle.
- **openclaw er plassholder** i Nivå 4 — innhold fylles inn senere.

## Konsekvenser
- Gjenbruker den visuelle malen fra `index.html` med minimal omskriving av rammeverket.
- Nivå 4 er fleksibelt: kan trimmes hardt hvis Nivå 1–3 renner over.
- To åpne punkter henger igjen: openclaw-payoff og om rtk er installert hos studentene.
